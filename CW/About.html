About This Project

3D Models

For my 3D App, I created three original 3D models in Blender: a teapot with cups, an iPod Nano, and a DJ booth with vinyls. I made all the textures myself, except for the vinyl labels and album cover images, which I sourced externally. Each model was exported as a .glb file to maintain material and animation compatibility.

Design

I designed the site, using images, gifs and fonts from the web to resemble a 2000s-era website, using CSS and Javascript. JavaScript handles most of the interactivity, including hover effects and audio playback for a retro tech vibe.

Integration

I used Three.js to display and animate the models. Audio clips were edited and synced to animations using CapCut, then embedded in the site. Additional images and text appear contextually based on the active model.

Interaction

The models can be rotated and zoomed using OrbitControls. JavaScript buttons allow users to toggle wireframe views, swap models, and trigger animations or sounds. These were part of the requirements. I added further interactions such as button hovers, and sounds designed to support the 2000s aesthetic.

Implementation

The project uses HTML5, CSS3, JavaScript, and Three.js. I organized assets into folders and tested functionality in Chrome and Firefox. Interactions are handled manually via JavaScript for flexibility and custom control.

Deeper Understanding

  • I created original 3D models from scratch and applied my own textures.
  • I built a fully custom UI with JavaScript and CSS.
  • I synced animation and sound externally in CapCut, then integrated them precisely.
  • I designed the entire experience to follow a nostalgic web aesthetic with purposefully styled interactions.

Publication & Testing

I have tested the zipped folder in Visual Studio Code and confirmed that the site loads correctly. All assets and scripts work as intended on the university's web server. I tested my 3D website in Chrome to make sure that the rendering of models, animations, and audio was working fine. There were testing sessions in week 11, where I only had a wireframe of the website. The feedback I received for the wireframe was that the layout was nice visually, so that is present throughout all of the webpages. During the testing that took place after, I received feedback that the layout felt a bit cramped, so I adjusted the sizes. Some testers also found the background colour too bright when combined with the text, so I tweaked the contrast and font weight to improve readability.
